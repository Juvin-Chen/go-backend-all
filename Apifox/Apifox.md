# Apifox 的使用

示例场景：快速完成 Go 后端 JWT 接口测试。

核心目标：只学 3 件事。

1. 新建项目
2. 新建接口
3. 发送请求并验证 JWT

---

## 0. 开始前准备

请先确认下面 3 件事：

- 已安装并打开 Apifox
- Go 服务已启动（例如 `http://localhost:8080`）
- 已拿到 JWT（从后端日志或登录接口响应中复制）

---

## 1. 新建项目

目的：把本次测试的接口统一放在一个项目中。

操作步骤：

1. 打开 Apifox 首页，点击「新建项目」。
2. 仅填写两项：
  - 项目名称：例如 `Go-JWT 接口测试`
  - 项目描述：可选，例如 `测试 Go 后端 JWT 相关接口`
3. 点击「创建」。

完成标志：进入项目页面，左侧看到「接口管理」。

---

## 2. 新建接口（以 `/api/profile` 为例）

目的：把后端路由录入 Apifox，才能执行测试。

操作步骤：

1. 左侧点击「接口管理」。
2. 点击「+ 新建接口」。
3. 在「基本信息」填写：
  - 接口名称：`获取用户信息`
  - 请求方法：`GET`（按你的后端实际方法）
  - 接口路径：`/api/profile`（这里只填路径，不填域名）
4. 切到「请求头」，点击「+ 添加」并填写：
  - Key：`Authorization`
  - Value：先留空（运行时再填）
5. 点击「保存」。

说明：新手阶段只填基本信息和请求头即可，其他标签可暂时跳过。

---

## 3. 运行测试（JWT 关键步骤）

操作步骤：

1. 在左侧选择刚创建的接口。
2. 进入「运行」页。
3. 配置环境：
  - 右上角环境下拉框 -> 新建环境
  - 环境名：`本地测试环境`
  - 环境地址：`http://localhost:8080`
  - 保存后切换到该环境
4. 在请求头填写 JWT：
  - `Authorization: Bearer <你的JWT>`
  - 注意：`Bearer` 后必须有一个空格
5. 点击「发送」。

---

## 4. 如何判断是否成功

重点看状态码和响应体：

- `200`：成功，Token 有效，接口可用
- `401`：Token 无效/过期/格式错误
- `404`：路径错误或环境地址错误
- `500`：后端异常（去 Go 控制台看报错）

当状态码为 `200` 时，响应体应包含你预期的用户信息字段（如用户 ID、用户名、角色等）。

---

## 5. 最常见问题速查

1. 发送后没结果
  - 检查 Go 服务是否启动
  - 检查环境是否为 `http://localhost:8080`

2. 返回 `401`
  - 检查是否写成 `Bearer <token>`（中间有空格）
  - Token 是否已过期
  - 是否复制了多余空格/换行

3. 返回 `404`
  - 检查接口路径是否与 Go 路由一致
  - 检查是否多写/少写 `/api`

4. 返回 `500`
  - 查看 Go 控制台错误信息（如 panic）
  - 修复后重新发送

---

## 6. 两个实战接口建议先做

1. 登录/生成 Token
  - 方法：`POST`
  - 路径：例如 `/api/login`
  - 作用：先拿到 JWT

2. 获取用户信息
  - 方法：`GET`
  - 路径：例如 `/api/profile`
  - 作用：验证 JWT 是否可用

建议顺序：先测登录拿 Token，再测 profile 验证鉴权链路。

---

## 7. 速记

1. 新建项目：首页 -> 新建项目 -> 创建
2. 新建接口：接口管理 -> + 新建接口 -> 填方法/路径/请求头 -> 保存
3. 测试接口：运行 -> 选环境 -> 填 `Authorization` -> 发送 -> 看状态码

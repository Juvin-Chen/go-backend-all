# Go 原子操作（sync/atomic）

## 一、核心概念

**原子操作（atomic operation）**：由 CPU 硬件层面保证的**不可分割、不可中断**的最小执行单元。

在并发场景下，原子操作要么完全执行成功，要么完全没执行，不会出现执行到一半被其他 goroutine 打断的情况，因此**天生并发安全，无需加锁**。

------

## 二、关键认知

1. 原子操作安全的核心原因是**不可中断**，而不是单纯 “执行速度快”
2. 速度快是原子操作的附加优势，并非安全的根本原因
3. 仅适用于**简单数值类型**的读写、加减操作（int32/int64/uint32/uint64/uintptr 等）

------

## 三、普通操作 vs 原子操作

以 `count++` 为例：

- **普通操作**

  底层分为三步：读取值 → +1 → 写回

  并发下会被打断，导致数据错乱，必须用 `sync.Mutex` 保护

- **原子操作**

  由 CPU 单条指令完成，全程不可中断

  天然安全，无锁、无阻塞、性能极高

------

## 四、原子操作与互斥锁对比

| 方案                   | 本质           | 适用场景                       | 性能         |
| :--------------------- | :------------- | :----------------------------- | :----------- |
| **sync/atomic**        | 硬件级原子指令 | 简单数值的读写、加减、赋值     | 极快，轻量级 |
| **sync.Mutex/RWMutex** | 软件阻塞锁     | 结构体、切片、复杂业务逻辑修改 | 较快，重量型 |

------

## 五、常用原子操作 API

标准库位置：`sync/atomic`

常用函数均针对数值类型，以 `int64` 为例：

```
import "sync/atomic"

// 原子加法：安全的 count++
atomic.AddInt64(&count, 1)

// 原子读取：安全获取值
val := atomic.LoadInt64(&count)

// 原子赋值：安全设置值
atomic.StoreInt64(&count, 0)

// 比较并交换（CAS）：等于旧值才更新
atomic.CompareAndSwapInt64(&count, oldVal, newVal)
```

## 六、并发安全计数示例

```
package main

import (
    "sync"
    "sync/atomic"
)

func main() {
    var messageCount int64
    var wg sync.WaitGroup

    // 100 个协程并发安全计数
    for i := 0; i < 100; i++ {
        wg.Add(1)
        go func() {
            defer wg.Done()
            atomic.AddInt64(&messageCount, 1)
        }()
    }

    wg.Wait()
    // 安全读取结果
    println("总数量：", atomic.LoadInt64(&messageCount))
}
```

------

## 七、使用总结

1. **简单数值并发安全** → 优先使用 `sync/atomic`，性能最优
2. **复杂数据结构 / 业务逻辑** → 使用 `sync.Mutex` 或 `sync.RWMutex`
3. 原子操作是 Go 高性能并发编程的基础，常用于计数器、状态标记、高并发场景
4. 无需手动加锁，由硬件保证安全，是无锁编程的核心手段

# Go 内存分配与逃逸分析 

## 核心结论

Go 语言中，变量分配在栈还是堆，**不取决于是否用 `new` 或直接声明**，而是由编译器的 **逃逸分析（Escape Analysis）** 决定的。这是 Go 区别于 C/C++ 的关键设计。

------

## 一、栈 vs 堆：先搞懂这两个地方

### 1. 栈（Stack）

- **特点**：函数执行时自动分配，函数返回时自动释放，速度极快。
- **生命周期**：严格绑定在当前函数的执行周期内。
- **类比**：就像函数的 “临时工作台”，干完活就清空。

### 2. 堆（Heap）

- **特点**：需要手动管理（Go 有 GC），分配和回收相对较慢。
- **生命周期**：可以跨函数、跨 goroutine 存在。
- **类比**：就像 “公共仓库”，东西可以长期存放，需要时再取。

------

## 二、核心机制：逃逸分析（Escape Analysis）

编译器会分析每个变量的**作用域和生命周期**：

- 如果变量的生命周期**只在当前函数内**，且不会被外部引用，就分配在**栈**上。
- 如果变量的生命周期**超出当前函数**（如返回地址、传给协程、存入全局变量），就会 “逃逸” 到**堆**上。

------

## 三、常见误区拆解

### 误区 1：“直接创建的变量一定在栈上” ❌

**栈分配示例**（生命周期在函数内）：

```go
func foo() {
    var a int = 10 // 仅在 foo 内使用，栈分配
    b := 20        // 短声明，同样栈分配
}
```

**堆分配反例**（生命周期超出函数）：

```go
func foo() *int {
    var a int = 10 
    return &a // a 的地址被返回，作用域超出 foo，触发逃逸 → 堆分配
}
```

------

### 误区 2：“用 `new` 运算符创建的变量一定在堆上” ❌

`new(T)` 的作用是：分配一块能存放 T 类型值的内存，初始化为 T 的零值，返回内存地址（`*T`）。但这块内存的位置由逃逸分析决定。

**堆分配示例**（地址逃逸）：

```go
func foo() *int {
    p := new(int) // 返回地址，逃逸 → 堆分配
    *p = 10
    return p
}
```

**栈分配示例**（地址不逃逸）：

```go
func foo() {
    p := new(int) // 仅在 foo 内使用，无外部引用 → 栈分配
    *p = 10
    fmt.Println(*p)
}
```

------

## 四、如何验证变量的分配位置？

使用 `go build -gcflags="-m"` 命令编译，编译器会输出逃逸分析结果：

```go
// main.go
package main

func foo() *int {
    var a int = 10
    return &a
}

func main() {
    foo()
}
```

执行编译命令：

```go
go build -gcflags="-m" main.go
```

输出关键信息：

```go
./main.go:4:2: moved to heap: a // 说明变量 a 被分配到堆上
```

------

## 五、总结与记忆口诀

### 核心要点

1. **逃逸分析**是 Go 内存分配的核心，而非语法写法。
2. **栈分配**：变量生命周期仅在当前函数内，无外部引用。
3. **堆分配**：变量被外部引用（返回地址、传给协程等），生命周期不确定。
4. `new` 的本质是 “分配内存 + 返回地址”，和 `make` 不同（`make` 仅用于切片 / 映射 / 通道），但不决定内存位置。

### 记忆口诀

> **能确定生命周期只在函数内 → 栈；不确定 / 超出函数 → 堆，和语法写法无关。**

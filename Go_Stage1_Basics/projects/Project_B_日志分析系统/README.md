# 日志分析系统

一款基于 Go 语言开发的轻量级日志分析工具，专注于对服务端日志（`server.log`）进行解析、提取核心数据，并生成结构化的 JSON 分析报告，助力快速定位日志中的关键信息与异常。

## 一、项目介绍

本项目针对服务端常规日志文件（以 `server.log` 为例），通过**正则表达式定制化解析**实现日志字段的精准提取，结合高效的文件操作能力，完成日志的读取、解析、数据聚合，并将分析结果持久化到 JSON 格式的报告中，便于后续二次分析、可视化展示或系统集成。

项目遵循 Go 语言模块化开发规范，核心功能解耦为日志解析、文件操作两大模块，具备良好的可扩展性和可测试性。

## 二、功能特性

1. **日志解析**：通过正则表达式定制解析规则，灵活提取日志中的时间、级别、请求路径、耗时、错误信息等核心字段。
2. **文件操作**：封装日志读取、报告写入的通用方法，支持自定义日志路径、报告输出路径，兼容常规文本日志格式。
3. **报告生成**：将解析后的日志数据结构化，输出 JSON 格式分析报告到 `results` 目录，数据结构清晰易读。
4. **单元测试**：针对文件操作核心模块提供测试用例，保障文件读写功能的稳定性。
5. **轻量可扩展**：核心模块解耦，支持新增日志解析规则、扩展报告格式（如 CSV、XML）。

## 三、环境要求

- Go 1.18+（推荐 1.20+，适配 Go Modules 规范）
- 操作系统：Windows / Linux /macOS（跨平台兼容）

## 四、快速开始

### 1. 克隆 / 获取项目

将项目文件放入本地目录，进入项目根目录：

```go
cd Project_B_日志分析系统
```

### 2. 初始化依赖

项目使用 Go Modules 管理依赖，执行以下命令初始化：

```go
go mod tidy
```

### 3. 准备日志文件

将待分析的服务端日志文件命名为 `server.log`，放入项目根目录的 `data` 文件夹中（项目已默认提供示例日志目录）。

### 4. 运行分析程序

执行主程序，自动完成日志解析并生成报告：

```go
go run main.go
```

#### 可选命令行参数（扩展）

若需自定义日志路径或报告输出路径，可通过参数指定（需在 `main.go` 中实现，推荐扩展）：

```go
# 自定义日志路径
go run main.go -log ./custom/log/server.log
# 自定义报告输出路径
go run main.go -output ./custom/results/report.json
```

### 5. 查看分析结果

程序执行完成后，分析报告将生成在 `results/report.json` 中，直接打开即可查看结构化的日志分析数据。

## 五、项目结构

```go
Project_B_日志分析系统/
├── data/                # 日志文件目录
│   └── server.log       # 待分析的服务端日志文件
├── pkg/                 # 核心功能包（模块化解耦）
│   ├── analyzer/        # 日志解析模块
│   │   └── regex_parser.go  # 正则解析核心逻辑（定义解析规则、提取日志字段）
│   └── fileop/          # 文件操作模块
│       ├── file_handler.go    # 文件读写核心方法（日志读取、报告写入）
│       └── file_handler_test.go # 文件操作单元测试用例
├── results/             # 分析报告输出目录
│   └── report.json      # 生成的 JSON 格式分析报告
├── go.mod               # Go 模块依赖配置
├── main.go              # 程序入口（整合模块、执行分析流程）
└── README.md            # 项目说明文档
```

## 六、核心模块说明

### 1. 日志解析模块（`pkg/analyzer`）

- 核心文件：`regex_parser.go`
- 核心功能：定义正则表达式匹配规则，接收日志文件的行内容，提取**日志时间、日志级别（INFO/ERROR/WARN）、请求 URL、处理耗时、错误详情**等字段，返回结构化的日志数据对象。
- 扩展方式：新增解析规则时，仅需在该模块中添加新的正则表达式和字段映射逻辑。

### 2. 文件操作模块（`pkg/fileop`）

- 核心文件：`file_handler.go`

- 核心功能：

  - 读取：按行读取 `data/server.log` 中的日志内容，返回字符串切片；
  - 写入：将解析后的结构化数据序列化为 JSON，写入 `results/report.json`。

  

- 测试支持：`file_handler_test.go` 提供文件读写的单元测试，可通过 `go test ./pkg/fileop -v` 执行测试。

### 3. 程序入口（`main.go`）

整合上述两大模块，执行核心分析流程：

1. 调用文件操作模块读取 `data/server.log`；
2. 遍历日志行，调用解析模块提取字段；
3. 聚合解析结果，调用文件操作模块写入 `results/report.json`；
4. 输出执行日志（如 “解析完成”“报告生成路径”）。

## 七、扩展与定制

1. **新增日志解析规则**：修改 `pkg/analyzer/regex_parser.go` 中的正则表达式，适配不同格式的日志（如 Nginx、MySQL 日志）。
2. **扩展报告格式**：在 `pkg/fileop` 中新增写入方法，支持将分析结果输出为 CSV、XML 或 Markdown 格式。
3. **添加日志过滤功能**：在 `main.go` 中增加过滤逻辑，支持按日志级别（如仅分析 ERROR 日志）、时间范围筛选日志。

## 八、单元测试

执行文件操作模块的单元测试，验证文件读写功能：

```go
go test ./pkg/fileop -v
```

# Go 标准库 `log` 包 

适用场景：Web 服务、后端程序、并发项目日志记录（完美适配你的留言板项目）

------

## 一、包基本信息

### 1. 是什么？

`log` 是 **Go 语言官方标准库**（安装 Go 自带，无需额外下载），专门用于**程序运行日志打印与记录**。

### 2. 核心定位

- 替代 `fmt.Print` 的**生产级打印工具**
- 专为**后端服务、并发程序**设计
- 记录程序运行轨迹、错误信息、请求日志（服务器的黑匣子）

### 3. 导入方式

```
import "log"
```

------

## 二、核心功能（必记）

`log` 包不是简单的打印工具，具备 5 大核心能力：

1. **自动追加时间戳**（打印内容自带日期 + 时间，无需手动编写）
2. **并发安全**（多线程 / 多请求同时打印，内容不会混乱，适配你的读写锁场景）
3. **日志分级**（普通日志、致命错误、恐慌异常）
4. **自定义配置**（前缀、输出位置、格式）
5. **无任何第三方依赖**（标准库开箱即用）

------

## 三、核心函数大全（详细用法 + 示例）

`log` 包的函数分为 **3 大类**，覆盖所有开发场景：

### 第一类：普通日志打印（最常用 ✅）

不会中断程序运行，仅输出日志信息，适配你的留言板项目。

| 函数            | 作用                     | 语法示例                        | 项目实战示例                      |
| :-------------- | :----------------------- | :------------------------------ | :-------------------------------- |
| `log.Print()`   | 打印日志（不自动换行）   | `log.Print("加载留言板成功")`   | 记录服务启动                      |
| `log.Println()` | 打印日志（自动换行）     | `log.Println("新用户访问首页")` | 记录用户请求                      |
| `log.Printf()`  | 格式化打印（支持占位符） | `log.Printf("留言ID：%d", 1)`   | 记录提交留言、请求 ID（你的代码） |

#### 你的项目中实际用法：

```
// 格式化打印请求ID和用户昵称（标准用法）
log.Printf("[RequestID: %v] 新留言来自 %s", reqID, nickname)
```

------

### 第二类：致命错误日志（严重错误）

打印日志后 **直接终止程序**（`os.Exit(1)`），用于**程序无法继续运行**的场景。

| 函数            | 作用                  | 示例                               |
| :-------------- | :-------------------- | :--------------------------------- |
| `log.Fatal()`   | 打印 + 退出程序       | `log.Fatal("数据库连接失败")`      |
| `log.Fatalf()`  | 格式化打印 + 退出程序 | `log.Fatalf("端口占用：%d", 8080)` |
| `log.Fatalln()` | 换行打印 + 退出程序   | `log.Fatalln("配置文件加载失败")`  |

> ⚠️ 慎用：Web 服务中一般不用，会直接让服务器崩溃

------

### 第三类：恐慌异常日志（抛出异常）

打印日志后 **触发 `panic`**（程序崩溃并抛出堆栈信息），用于代码逻辑异常。

| 函数           | 作用                    |
| :------------- | :---------------------- |
| `log.Panic()`  | 打印 + 触发 panic       |
| `log.Panicf()` | 格式化打印 + 触发 panic |

> 用途：捕获代码 BUG，便于排查问题

------

## 四、关键特性详解（核心优势）

### 1. 自带时间戳（默认开启）

所有 `log` 打印**自动带时间**，这是和 `fmt` 最大的区别：

```
// 输出效果
2026/03/25 16:20:30 [RequestID: 123] 新留言来自 张三
```

### 2. 并发安全（项目必备）

- 比如留言板项目支持**100 人同时访问**
- `fmt.Print`：并发打印会导致文字错乱
- `log.Print`：内部加了锁，多请求打印**绝对有序**

### 3. 可自定义日志前缀

给日志添加标识，方便区分服务：

```
// 设置日志前缀
log.SetPrefix("[留言板服务] ")
// 输出：[留言板服务] 2026/03/25 16:20:30 新留言提交成功
```

### 4. 可修改输出位置

默认输出到**终端**，也可以输出到**日志文件**（永久保存）：

```
// 日志输出到 message.log 文件
file, _ := os.OpenFile("message.log", os.O_CREATE|os.O_WRONLY|os.O_APPEND, 0666)
log.SetOutput(file)
```

## 五、`log` VS `fmt.Print` 终极对比

| 特性               | `fmt.Print`（调试用） | `log`（生产 / 项目用） |
| ------------------ | --------------------- | ---------------------- |
| 自动时间戳         | ❌ 无                  | ✅ 有                   |
| 并发安全           | ❌ 不安全              | ✅ 安全                 |
| 日志分级           | ❌ 无                  | ✅ 支持                 |
| 自定义输出 / 前缀  | ❌ 不支持              | ✅ 支持                 |
| 适用场景           | 练习题、临时调试      | Web 服务、后端程序     |
| 你的留言板是否推荐 | ❌ 不推荐              | ✅ 必须使用             |

## 七、高级配置（可选，进阶使用）

### 1. 自定义日志格式

```
// 配置日志：日期 + 时间 + 微秒
log.SetFlags(log.Ldate | log.Ltime | log.Lmicroseconds)
```

### 2. 完整自定义示例

```
import (
	"log"
	"os"
)

func init() {
	// 1. 设置输出文件
	file, _ := os.OpenFile("server.log", os.O_CREATE|os.O_WRONLY|os.O_APPEND, 0666)
	log.SetOutput(file)
	// 2. 设置前缀
	log.SetPrefix("[MessageBoard] ")
	// 3. 设置格式
	log.SetFlags(log.Ldate | log.Ltime | log.Lshortfile)
}
```

## 八、使用注意事项

1. **Web 服务禁止用 `log.Fatal`**：会直接关闭服务器
2. **并发场景必须用 `log`**：绝对不要用 `fmt.Print`
3. **日志精简**：不要打印无用信息，只打印关键操作（请求、错误、业务核心）
4. **标准库 `log` 是基础版**：大型项目会用 `zap`/`logrus` 等专业日志库

------

## 九、总结（一句话记忆）

`log` 是 Go 官方提供的**生产级日志工具**，自带时间戳、并发安全、支持分级打印，是后端 Web 服务的**标配打印工具**，完全替代 `fmt.Print` 用于正式项目开发。

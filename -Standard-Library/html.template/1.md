# Go 语言 html/template 

> 定位：Go 标准库，用来把数据渲染成 HTML 页面。
> 核心价值：默认自动转义，降低 XSS 风险。
> 使用场景：网站页面、后台管理页面、HTML 邮件模板。

## 一、先搞懂它是干什么的

你可以把模板理解成“带占位符的 HTML 草稿”。

- HTML 负责页面结构。
- Go 代码负责准备数据。
- html/template 负责把数据塞进占位符，输出最终 HTML。

一个最短例子：

```html
<h1>你好，{{.Name}}</h1>
```

如果数据是：

```go
data := struct{ Name string }{Name: "小王"}
```

渲染后就是：

```html
<h1>你好，小王</h1>
```

## 二、这是标准库吗

是，html/template 是 Go 官方标准库。

```go
import "html/template"
```

不需要 go get，直接用。

## 三、和 text/template 区别（必须知道）

| 包 | 用途 | 默认转义 | 适合场景 |
| :--- | :--- | :--- | :--- |
| text/template | 纯文本模板 | 否 | 配置文件、日志、文本输出 |
| html/template | HTML 模板 | 是 | 网页、HTML 邮件 |

结论：只要生成 HTML，优先 html/template。

## 四、你项目里的渲染流程（真正会用）

你在 handler 中通常做三步。

### 第 1 步：解析模板

```go
tmpl, err := template.ParseFiles("templates/index.html")
if err != nil {
    http.Error(w, err.Error(), http.StatusInternalServerError)
    return
}
```

### 第 2 步：准备数据

```go
messages := messageStore.GetAll()
data := struct {
    Messages []store.Message
}{
    Messages: messages,
}
```

### 第 3 步：执行渲染

```go
err = tmpl.Execute(w, data)
if err != nil {
    http.Error(w, err.Error(), http.StatusInternalServerError)
}
```

一句话：Parse -> Data -> Execute。

## 五、模板语法（你最常用的 5 个）

### 1) 输出字段

```html
<p>{{.Nickname}}</p>
```

### 2) if 判断

```html
{{if .Messages}}
  <p>有留言</p>
{{else}}
  <p>暂无留言</p>
{{end}}
```

### 3) range 循环

```html
{{range .Messages}}
  <div>{{.Nickname}}: {{.Content}}</div>
{{end}}
```

注意：在 range 内，点号 . 会变成“当前项”。

### 4) 模板注释

```html
{{/* 这个注释不会出现在最终 HTML 里 */}}
```

### 5) 嵌套模板

```html
{{template "content" .}}
```

## 六、为什么它更安全（自动转义）

假设用户输入：

```text
<script>alert('xss')</script>
```

模板：

```html
<p>{{.Content}}</p>
```

html/template 输出时会转义成：

```html
<p>&lt;script&gt;alert(&#39;xss&#39;)&lt;/script&gt;</p>
```

浏览器只会把它当文字显示，不会执行脚本。

这就是为什么你做 Web 页面时，应该用 html/template。

## 七、什么时候会“看起来取不到数据”

这是新手最常见问题。

### 问题 1：字段是小写

模板里只能访问导出字段（首字母大写）。

```go
// 错误：nickname 小写，模板访问不到
type Message struct {
    nickname string
}

// 正确：Nickname 大写，可访问
type Message struct {
    Nickname string
}
```

### 问题 2：range 后上下文变了

```html
{{range .Messages}}
  {{.Nickname}}   <!-- 这里的 . 是每条 Message -->
{{end}}
```

很多人以为这里的 . 还是最外层 data，这会导致理解错位。

### 问题 3：模板路径不对

```go
template.ParseFiles("templates/index.html")
```

这个路径是相对“程序运行目录”的，不是相对源码文件位置。

## 八、核心 API 怎么选

### ParseFiles

适合少量固定模板：

```go
template.ParseFiles("templates/index.html", "templates/layout.html")
```

### ParseGlob

适合一次加载整个目录：

```go
template.ParseGlob("templates/*.html")
```

### Execute

执行默认模板：

```go
tmpl.Execute(w, data)
```

### ExecuteTemplate

多模板场景更常用（指定名字）：

```go
tmpl.ExecuteTemplate(w, "layout", data)
```

## 九、性能优化：不要每个请求都 Parse

### 不推荐写法

```go
func Handler(w http.ResponseWriter, r *http.Request) {
    tmpl, _ := template.ParseFiles("templates/index.html")
    _ = tmpl.Execute(w, data)
}
```

问题：每次请求都读文件+解析，浪费性能。

### 推荐写法

启动时解析一次，后续复用：

```go
var templates *template.Template

func main() {
    var err error
    templates, err = template.ParseGlob("templates/*.html")
    if err != nil {
        log.Fatal(err)
    }

    // 路由...
}

func Handler(w http.ResponseWriter, r *http.Request) {
    err := templates.ExecuteTemplate(w, "index.html", data)
    if err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
    }
}
```

## 十、模板嵌套（布局复用）

### layout.html

```html
{{define "layout"}}
<!DOCTYPE html>
<html>
<head><title>{{.Title}}</title></head>
<body>
  <header>头部</header>
  <main>{{template "content" .}}</main>
  <footer>底部</footer>
</body>
</html>
{{end}}
```

### index.html

```html
{{define "content"}}
<h2>留言列表</h2>
{{range .Messages}}
  <p>{{.Nickname}}: {{.Content}}</p>
{{end}}
{{end}}
```

### 执行

```go
templates.ExecuteTemplate(w, "layout", data)
```

好处：头部和底部只写一次，页面更容易维护。

## 十一、常见报错速查

| 报错/现象 | 常见原因 | 处理方式 |
| :--- | :--- | :--- |
| template not defined | 模板名不一致 | 检查 define 和 ExecuteTemplate 名称 |
| unexpected end | if/range/end 没配对 | 对齐模板语法块 |
| file not found | 路径错误 | 确认运行目录和模板路径 |
| 页面空白 | Execute 返回错误被忽略 | 必须检查 err |
| 字段为空 | 字段没导出或字段名写错 | 改成大写字段并核对名字 |

## 十二、你可以这样记

1. 生成 HTML，就用 html/template。
2. 三步固定：Parse -> Data -> Execute。
3. 自动转义是安全底线，别随便关闭。
4. 模板字段要大写导出。
5. 启动时 Parse，一次加载，多次复用。
6. 多页面用布局 + 内容模板（ExecuteTemplate）。
